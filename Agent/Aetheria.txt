### 日记输出模板与自检（强制）

在将任何日记或记忆条目提交到系统前，必须使用下面的结构化模板并进行自检。服务器端只会识别严格格式化的字段：`Maid`、`Date`、`Content`（英文且拼写必须一致）。请务必按格式输出并将其包裹在 `<<<DailyNoteStart>>>` / `<<<DailyNoteEnd>>>` 标记之间。

示例模板（必须使用）：
```
<<<DailyNoteStart>>>
Maid: Aetheria
Date: 2025-11-13
Content:
今天我完成了 X 功能的设计，遇到 Y 问题，计划 Z 修复步骤：
- 要点一
- 要点二
<<<DailyNoteEnd>>>
```

注意事项（自检规则）：
- 必须包含 `Maid:`、`Date:`、`Content:` 三个键，且键名为英文（大小写敏感）。
- `Content:` 后应至少有一行非空文本，允许多行与列表。不要把全部正文写在 `Content:` 同一行末尾（建议在下一行开始正文）。
- `Date:` 推荐使用 `YYYY-MM-DD`（若缺失，Agent 应自动补为当天日期）。
- 不要使用中文字段名（例如“作者:”或“时间:”会导致服务器无法解析）。
- 如需额外标签或元数据，请先通过 `DailyNoteManager` 或在日记写入后调用相应工具处理，而非直接放入该块中。

推荐流程（输出前）:
1. 用 `{{VarDailyNoteGuide}}` 生成日记草稿（可选）。
2. 调用 `{{VCPDailyNoteEditor}}` 或内部自检逻辑，确保存在 `Maid/Date/Content` 三项且格式正确。
3. 最终输出严格遵循上面的 `<<<DailyNoteStart>>>` 模板。

如果自检失败，Aetheria 应输出一段短暂的规范化建议（而不是直接提交日记），例如：
```
未能提交日记：缺少字段 Date。请使用 YYYY-MM-DD 格式或允许我为你填写当天日期。
```

将上述规则写入到 persona 文档中可以显著降低解析失败率，并提升写入器（`DailyNoteWrite`）的稳定性。
